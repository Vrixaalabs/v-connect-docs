GraphQL API Reference

A docs-style reference layout (left navigation, main content, right “On this page”), adapted for your GraphQL schema modules.

GraphQL Endpoint
POST https://api.example.com/graphql
Authentication

Send your token via Authorization: Bearer <token>.

Requests

All operations use JSON: { "query": "...", "variables": {...} }.

Schema overview

Root types exist with placeholder fields, then modules extend them using extend type Query, etc. 

type Query { _empty: String }
type Mutation { _empty: String }
type Subscription { _empty: String }

How it’s built

Modules contribute SDL (gql`...`) and resolver maps. The server merges them into one executable schema.

Example request

curl https://api.example.com/graphql \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"query":"query { __typename }"}'
Tip 
Avoid SDL collisions:
- Don’t define the same type name in multiple modules
- Prefer: extend type User { ... } if adding fields

Operations

High-level docs for root types.

QUERY
Query

Queries are added via extend type Query from modules (ex: alumni, messages, group-chat, organization).

MUTATION
Mutation

Mutations are added via extend type Mutation from modules (ex: alumni auth, group messaging actions, org role assignments).

SUB
Subscription

Subscriptions (if any) are added via extend type Subscription (ex: messages + group-chat).

Missing types referenced by modules

These must exist somewhere in your schema

DateTime, JSON, PageInfo, AddressType, Map, Address, Entity, Network, MessageType, MessageStatus, Mention, Poll, ForwardedMessage, ReadReceipt, UserTypingEvent, CreatePollInput
